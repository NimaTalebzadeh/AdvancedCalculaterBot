# AdvancedCalculaterBot CAS Refactor Plan

> Goal: Replace brittle string/regex symbolic handling with a generalized AST-driven symbolic engine backed by AngouriMath.

## Current Problems

The current system mixes:
- regex parsing
- string rewrites
- special-case overrides
- partial AngouriMath delegation

This causes:
- expression explosions
- endless hardcoded exceptions
- weak simplification
- incorrect equivalence handling
- timeout-heavy symbolic operations

## Refactor Strategy

The migration must be incremental.

We cannot delete all hardcoded rules immediately because they currently preserve correctness.

The safe path is:
1. build a stable AST/AngouriMath core
2. route more operations into it
3. compare outputs against regression tests
4. gradually delete hardcoded handlers

---

# Phase 1 — Stabilize AST / AngouriMath Core

## Objective

Create a unified symbolic execution layer based primarily on AngouriMath Entities instead of string manipulation.

## Tasks

### Task 1.1 — Create SymbolicEngine service

Create:
- Services/Mathematics/Symbolic/SymbolicEngine.cs

Responsibilities:
- parse expressions into AngouriMath Entity
- canonical formatting
- simplification
- derivative
- integral
- limit
- factor
- expand
- series

API:

```csharp
public static class SymbolicEngine
{
    public static Entity Parse(string expr);
    public static string Simplify(string expr);
    public static string Differentiate(string expr, string variable, int order = 1);
    public static string Integrate(string expr, string variable);
    public static string Factor(string expr);
    public static string Expand(string expr);
}
```

Verification:
- parse x^2+1
- derivative of sin(x)
- integral of x^2

---

### Task 1.2 — Add canonical formatter

Create:
- Services/Mathematics/Symbolic/SymbolicFormatter.cs

Goals:
- remove AngouriMath noise
- normalize powers
- normalize multiplication
- normalize domain annotations

Examples:
- x ^ 2 → x^2
- 2 * x → 2*x

Verification:
- formatting snapshot tests

---

### Task 1.3 — Add AST-based simplification pipeline

Replace string simplification flow with:

```csharp
Entity expr = MathS.FromString(expression);
expr = expr.Simplify();
```

Then format via SymbolicFormatter.

Verification:
- simplify((x^3+3*x^2+3*x+1))
- simplify((x^4-1)/(x^2-1))

Include:
